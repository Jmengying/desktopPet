# 二次元桌面宠物 — 设计文档

## 1. 概述

开发一款适配 Windows 10/11 的二次元桌面宠物软件。默认角色「白玉」为白发猫娘萝莉风格。宠物常驻桌面，支持拖拽、点击互动、鼠标跟随、桌面走动、随机事件等功能。

### 技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| 框架 | Electron | Web 生态丰富，透明窗口和动画实现成熟 |
| 动画方案 | SVG/Canvas 代码绘制 + Lottie 扩展 | 默认角色用代码绘制，后续可导入 Lottie JSON |
| 角色风格 | 标准二次元（白发猫娘萝莉） | 参考萌娘百科「白玉」形象 |
| 架构 | 多窗口（Pet Window + Config Window） | 完全隔离，宠物窗口不受 UI 干扰 |

### MVP 范围（基础完整版）

- 透明置顶 + 点击穿透
- 拖拽移动 + 点击互动
- 右键菜单（磨砂玻璃风格）
- 待机动画 + 随机事件
- 鼠标跟随/注视
- 桌面走动（可开关）
- 装扮切换
- 音效系统
- 开机自启

## 3. 角色设计 — 白玉

### 3.1 外观特征

**基础形象**
- 发型：长直白发，齐刘海
- 发色：纯白色 / 银白色
- 猫耳：白色猫耳，耳内粉色
- 眼睛：大眼睛，蓝色/紫色瞳孔
- 身材：Q版萝莉比例（头身比约 1:2.5）
- 表情：腮红明显，嘴角微翘

**服装 & 配饰**
- 上衣：白色水手服 / 连衣裙
- 配色：白+粉 / 白+蓝 为主
- 领结：蝴蝶结或缎带
- 尾巴：白色猫尾巴
- 袜子：白色过膝袜
- 鞋子：小皮鞋 / 玛丽珍鞋

### 3.2 SVG 图层结构

```
Layer 0:  Shadow (投影)
Layer 1:  Tail (尾巴) ← spring 物理模拟
Layer 2:  Body (身体轮廓 + 服装)
Layer 3:  Left Arm (左手)
Layer 4:  Head (头部)
Layer 5:  Hair Back (后发)
Layer 6:  Face (五官)
Layer 7:  Hair Front (前发/刘海)
Layer 8:  Cat Ears (猫耳) ← 可独立抖动
Layer 9:  Right Arm (右手)
Layer 10: Accessories (配饰/蝴蝶结)
Layer 11: Effects (腮红/高光/粒子)
```

每层使用 SVG `<g>` 分组，transform 控制位移/旋转。

### 3.3 表情系统

通过 SVG path 插值实现表情过渡，眼睛/嘴巴/腮红独立控制：

| 表情 | 触发场景 |
|------|----------|
| 默认微笑 ◕‿◕ | 待机状态 |
| 开心 ◕ω◕ | 点击互动 |
| 困倦 ◕_◕ | 闲置过久 |
| 惊讶 ◕‸◕ | 鼠标快速划过 |
| 眯眼笑 ◡‿◡ | 被摸头 |
| 害羞 ◕ᴗ◕ | 鼠标触碰 |
| 睡觉 —‿— | 长时间闲置 |
| 好奇 ◕◇◕ | 鼠标靠近 |

## 4. 动画系统

### 4.1 动画状态机

```
                    ┌──────────────┐
                    │   IDLE       │ ◄─────────────────────┐
                    │  (站立待机)   │                       │
                    └──────┬───────┘                       │
                           │                               │
              ┌────────────┼────────────┐                  │
              ▼            ▼            ▼                  │
      ┌───────────┐ ┌───────────┐ ┌───────────┐          │
      │ WALK      │ │ LOOK      │ │ STRETCH   │          │
      │ (踱步)    │ │ (张望)    │ │ (伸懒腰)  │          │
      └─────┬─────┘ └─────┬─────┘ └─────┬─────┘          │
            │             │             │                  │
            └─────────────┼─────────────┘                  │
                          ▼                                │
                   ┌──────────────┐                        │
                   │   IDLE       │ ──────────────────────┘
                   └──────────────┘
```

**触发事件：**
- click → BLINK (眨眼) / WAVE (挥手)
- drag → DRAGGED (被拖拽表情)
- hover → TURN_HEAD (转头注视)
- idle_long → SLEEP (打瞌睡) / YAWN (打哈欠)
- random → JUMP (小跳) / PEEK (左右张望)

### 4.2 实现方式

- 骨骼系统：SVG transform 链式变换
- 表情：眼睛/嘴巴路径插值
- 物理：头发/裙摆/尾巴 spring 模拟
- 粒子：Canvas 2D 叠加层
- 缓动：requestAnimationFrame

无需外部依赖，纯 JS 实现，后续可导入 Lottie JSON 替换。

### 4.3 性能优化

- 脏区域检测，仅重绘变化部分
- 离屏 Canvas 缓存静态部件
- 非活跃时降低帧率（30→10fps）
- 休眠模式：完全停止渲染
- SVG 路径数据复用
- 按需加载动画片段
- GC 友好的对象池

## 5. 交互系统

### 5.1 鼠标交互逻辑

```
mousedown (左键)
    ├── 在角色身体区域?
    │     ├── YES → 记录偏移量, 进入 DRAG_WAIT 状态
    │     └── NO  → 忽略 (点击穿透)

mousemove
    ├── DRAG_WAIT + 移动距离 > 5px?
    │     └── YES → 进入 DRAGGING 状态, 更新窗口位置

mouseup
    ├── DRAGGING? → 结束拖拽, 回到 IDLE
    └── DRAG_WAIT (未移动)? → 触发 CLICK 事件
          └── 播放随机互动动画 + 音效
```

使用 `setIgnoreMouseEvents(true, { forward: true })` 实现非角色区域穿透，同时保留 mousemove 事件用于检测鼠标靠近。

### 5.2 鼠标跟随效果

**注视跟踪**
- 鼠标在角色附近时，眼睛跟随鼠标方向
- 头部轻微转向鼠标位置
- 超出范围后缓慢回正
- 使用 atan2 计算角度，lerp 平滑过渡

参数：
```
followRadius: 200px
eyeMaxAngle: 15°
headMaxAngle: 8°
lerpSpeed: 0.08
```

**靠近反应**
- 鼠标进入 100px 范围 → 转头 + 好奇表情
- 鼠标触碰角色 → 害羞/眯眼表情
- 鼠标快速划过 → 惊讶表情
- 鼠标离开 → 微笑 + 挥手

参数：
```
hoverRadius: 100px
touchRadius: 30px
reactionCooldown: 2s
```

### 5.3 右键菜单

磨砂玻璃风格，直接在宠物窗口内弹出（非系统菜单）。

样式：
```css

backdrop-filter: blur(20px);
border: 1px solid rgba(255, 255, 255, 0.5);
border-radius: 14px;
box-shadow: 0 8px 32px rgba(0, 0, 0, 0.08);
```

菜单项：

2. 🎀 装扮设置
3. ---
4. 🔊 音效开关 (ON/OFF)
5. 🖱️ 鼠标跟随 (ON/OFF)
6. ✨ 随机动作 (ON/OFF)
7. 🚶 桌面走动 (ON/OFF)
8. ---
9. 🚀 开机自启 (ON/OFF)
10. ---

## 6. 桌面走动功能

### 6.1 走动状态机

```
IDLE (站立)
  │
  ├── 随机定时器到期 (30s~120s)
  │     └── 选择目标方向 → WALK_START
  ▼
WALK_START (起步) → 转身朝向目标 → WALKING
  ▼
WALKING (行走中)
  ├── 到达目标点? → WALK_STOP
  ├── 到达屏幕边缘? → WALK_STOP (转身)
  ├── 被用户拖拽? → IDLE (中断)
  └── 用户关闭走动? → IDLE
  ▼
WALK_STOP (停步)
  ├── 70% → IDLE (站立一会儿)
  ├── 20% → WALK_START (继续走)
  └── 10% → 特殊动作 (坐下/趴下/伸懒腰)
```

### 6.2 走动参数

```
walkSpeed: 1~3 px/frame (缓慢漫步)
walkDistance: 100~400 px (随机目标距离)
idleDuration: 5~30s (站立停留时间)
direction: 左/右 随机选择
```

### 6.3 走动动画

- 起步：转身 + 轻微前倾
- 行走：左右交替踏步 + 身体微晃
- 头发：随行走节奏摆动 (spring)
- 尾巴：跟随身体摇摆
- 猫耳：轻微上下抖动
- 停步：减速 + 站稳 + 回正

特殊行为：小跑（10%概率）、踱步、坐下休息、趴下、追鼠标、避开边缘。

### 6.4 窗口位置方案

推荐方案：Pet Window 全屏透明 + 角色在窗口内移动。避免频繁 `setPosition` 开销。

## 7. 随机事件系统

### 7.1 事件触发机制

```
RandomEventScheduler:
  interval: 30s ~ 120s (随机)
  cooldown: 10s (防连续触发)
```

| 事件 | 权重 | 条件 |
|------|------|------|
| 打哈欠 | 20 | 闲置>30s |
| 打瞌睡 | 15 | 闲置>60s |
| 挥手打招呼 | 25 | 无 |
| 左右张望 | 20 | 无 |
| 小跳玩耍 | 15 | 无 |
| 伸懒腰 | 10 | 闲置>45s |
| 玩尾巴 | 10 | 无 |

加权随机选择，权重越高越容易触发。

### 7.2 状态联动

**电脑闲置检测（每5秒）**
- 无键鼠操作 > 60s → 触发慵懒、打瞌睡
- 无键鼠操作 > 300s → 触发睡觉动画 + 降低帧率至 10fps

**活跃状态检测**
- 频繁键鼠操作 → 触发好奇张望
- 窗口切换 → 转头看新窗口

使用 Electron powerMonitor API + 全局键鼠钩子检测。
